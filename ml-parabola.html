Learning ML #1: Teaching a Neural Net to Fit a Parabola

April 6, 2026
10 min read
By Declan Porter

I decided to learn machine learning properly — not by copying tutorials, but by building everything from scratch with an LLM as my teacher. This is the first project, training a small neural network to fit y = x².

Why a Parabola?

It's the simplest nonlinear regression problem you can construct. One input, one output, a curve a human could draw in two seconds.

The Dataset

I sampled 2000 random x values uniformly in [−1, 1], computed y = x², and split 80/20 into training and validation sets. Nothing fancy. The important thing I learned immediately: PyTorch's nn.Linear expects 2D tensors of shape [N, features], not 1D arrays.

So a list of 1600 scalars needs to become a tensor of shape [1600, 1]. The fix is reshape(-1, 1). The -1 means "figure out this dimension for me" — PyTorch divides the total elements by 1 and gets 1600 rows automatically. It's not in-place, so you need assignment: 

trainingTensor = trainingTensor.reshape(-1, 1)

The Model

A single Linear(1, 1) can only learn a straight line. A parabola is nonlinear, so I needed hidden layers with nonlinear activations between them. I went with: 

class FirstNN(nn.Module):
    def __init__(self):
        super(FirstNN, self).__init__()
        self.input = nn.Linear(1, 32)
        self.input_act = nn.Tanh()
        self.hidden = nn.Linear(32, 32)
        self.hidden_act = nn.Tanh()
        self.output = nn.Linear(32, 1)

    def forward(self, x):
        l1 = self.input_act(self.input(x))
        l2 = self.hidden_act(self.hidden(l1))
        l3 = self.output(l2)
        return l3

Width 32 in the hidden layers, Tanh activations between them, no activation on the output (important — Tanh squashes to [−1, 1], but our targets can be anything in [0, 1]). Loss function: nn.MSELoss, the mean squared error between predicted and true y.

The Training Loop

The most important thing I learned in this project is the exact order of operations inside a training loop:

  1. Zero gradients — PyTorch accumulates gradients by default, so you clear old ones first.
  2. Forward pass — run inputs through the model to get predictions.
  3. Compute loss — the single number that tells the model how wrong it is.
  4. Backward pass — compute gradients ∂loss/∂θ for every parameter.
  5. Optimizer step — update the parameters using those gradients.
for epoch in range(1000):
    optimizer.zero_grad()
    preds = model(training_x_tensor).reshape(-1)
    train_loss = loss(preds, training_y_tensor)
    train_loss.backward()
    optimizer.step()

    if epoch % 10 == 0:
        with torch.no_grad():
            val_preds = model(validation_x_tensor).reshape(-1)
            val_loss = loss(val_preds, validation_y_tensor)
            print(f"train_loss={train_loss:.4f} | val_loss={val_loss:.4f}")

Optimizers: SGD vs Adam vs AdamW

I went deep on optimizers here. SGD is the conceptual foundation — you move parameters in the direction that reduces loss:

θt+1 = θt − η · ∇L(θt)

Adam is fancier. It keeps two running averages per parameter: m (first moment — the direction) and v (second moment — the magnitude squared). These let Adam adapt the step size per parameter individually. Parameters with large, unstable gradients get smaller steps; parameters with small gradients get relatively larger ones.

AdamW is Adam plus decoupled weight decay. Instead of mixing the decay into the gradient before the adaptive scaling (like regular Adam + L2), AdamW applies a direct shrink each step:

θt+1 = (1 − η·λ) · θt − η · (Adam update term)

I tested all three. Interestingly, SGD converged much more smoothly on this simple problem — AdamW sat near loss ~86 for a while before diving down. The conclusion: adaptive methods have warm-up overhead (their moment estimates take a few steps to become reliable), which can look like a plateau on simple problems.

Results

After 1000 epochs with Adam at lr=0.001: 

train_loss=0.0002 | val_loss=0.0002

The model learned the parabola. Train and validation losses tracking together (no divergence) confirmed no overfitting. An RMSE of ~0.014 on a y range of [0, 1] is excellent.

What I Learned

  • Tensor shapes are the #1 source of bugs in PyTorch. Print shapes constantly early on.
  • The training loop order — zero_grad, forward, loss, backward, step — is sacred. Never swap backward and step.
  • Always use nn.Module. It gives you parameter management, train/eval modes, state dict saving, and cleaner code for free.
  • Don't put an activation on your regression output — it will clip your predictions.
  • Adam's "bias correction" is about correcting cold-start underestimation of its moment estimates, not about layer biases.
